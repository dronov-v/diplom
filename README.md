# Рекомендательная система для интернет-магазина электроники

Дипломный проект по разработке и сравнению рекомендательных алгоритмов для интернет-магазина электроники на данных о поведении пользователей.

## Описание

Проект строит рекомендации на основе неявной обратной связи: просмотров, добавлений в корзину и покупок. События преобразуются в числовой вес взаимодействия, после чего на подготовленных данных обучаются и сравниваются несколько моделей.

В работе используются:

- фильтрация исходного датасета по категории электроники;
- формирование взвешенных взаимодействий пользователь-товар;
- подготовка общей выборки для честного сравнения моделей;
- обучение и оценка ALS, SVD++ и SLIM;
- построение графиков с результатами сравнения.

## Алгоритмы

- **Implicit ALS** - матричная факторизация для неявной обратной связи.
- **SVD++** - модель на условных оценках, сформированных из веса взаимодействия.
- **SLIM** - item-item модель, которая строит связи между товарами по пользовательскому поведению.

Папка `data/` не хранится в репозитории, потому что исходный и промежуточные CSV-файлы занимают много места.

## Данные

Исходный датасет: **eCommerce Behavior Data from Multi-Category Store**.

Для запуска полного пайплайна нужно поместить файл:

```text
data/2019-Nov.csv
```

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

Скрипты запускаются последовательно:

```bash
python src/01_check_dataset.py
python src/02_filter_electronics.py
python src/03_build_interactions.py
python src/04_create_matrix.py
python src/05_prepare_common_data.py
python src/06_ALS.py
python src/07_SVDplusplus.py
python src/08_SLIM.py
python src/09_compare_models.py
python src/10_draws.py
```

## Результаты

Итоговые графики сохраняются в папку `results/figures/`. Табличные промежуточные результаты не добавляются в Git и воспроизводятся повторным запуском пайплайна.
